# Coding Style Guidelines

1.  **Clarity over Cleverness**: Write code that is easy to read and understand.
2.  **Naming Conventions**: Use descriptive names for variables and functions.
    *   Python: `snake_case`
    *   JavaScript/TypeScript: `camelCase`
    *   Go: `PascalCase` (exported), `camelCase` (internal)
3.  **Comments**: Comment *why*, not *what*. Explain complex logic.
4.  **Formatting**: Adhere to the project's linter/formatter (Prettier, Black, Gofmt).
